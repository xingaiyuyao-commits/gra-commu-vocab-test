# 単語テスト再実装（デザイン刷新）設計

日付: 2026-07-19

## 背景と目的

トップページの「単語テスト」は現在、外部のRailwayアプリ
（realtime-chat-production-b389.up.railway.app）へのリンクだが、そのソースコードが
手元に存在せず、デザイン修正ができない。そこで**同じゲームルールのまま**このプロジェクト内に
再実装し、デザイン・使い勝手を改善する。

- ゲームルールは元と同じ: 穴埋め入力式・自分のペースで5問→提出・全員提出後にランキング発表
- 変えるのは見た目と操作感のみ（スマホ最適化、コインタワー/並べ替えバトルとデザイン統一）
- 問題は手持ちのPDF（デスクトップ「単語テスト」フォルダ）から取り込む

## 問題データ: `wordtests.js`

PDF 18ファイルから抽出して静的データファイルを生成する。

- 収録: Clacel 2.0（Series 1〜8）、IELTS（Series 1〜5）、TOEIC（Series 1〜5）＝18セット
- 英検は収録しない（ユーザー選択）
- 各セット20語（例外: Clacel Series 8のみ元PDFが12語構成のため12語）。
  PDFのWord Listにある例文の対象単語を `___` に置換して問題化する

```js
module.exports = {
  clacel: {
    label: "Clacel 2.0",
    series: [
      { name: "Series 1", items: [
        { sentence: "The children ___ to be superheroes.", answer: "pretended",
          base: "pretend", hint: "p________", ja: "ふりをする" },
        // ×20
      ] },
      // ×8
    ],
  },
  ielts: { ... },  // ×5
  toeic: { ... },  // ×5
};
```

- `answer` は例文中の活用形そのまま（pretended, supplies等）を小文字で保持。
  判定は既存コインタワーと同じ「両者を小文字化して完全一致」
- `hint` は頭文字＋文字数分のアンダースコア
- `ja` はPDFの日本語の意味（単語の意味）を使用

## サーバー: `server.js` に `quiz:*` イベントを追加

既存2ゲームと同じパターン（独立した rooms オブジェクト＋Socket.IOイベント群）。

- `quiz:createRoom { name }` → 4文字ルームコード発行、ロビーへ
- `quiz:joinRoom { roomCode, name }` → ロビー中のみ参加可
- `quiz:startGame { category, seriesIndex }` → ホストのみ。選択セットの20語から
  ランダム5問を抽選し、**全員に同じ5問**を配布。phase を playing に
- `quiz:submit { answers: [5つの文字列] }` → サーバー側で採点し、提出時刻を記録。
  全員に提出状況（n/総人数）をブロードキャスト
- 全員提出したら `quiz:results` をブロードキャスト:
  順位は 正解数の降順 → 提出の早さ順。各問の正答も送り、答え合わせ表示に使う
- 切断処理: 既存と同様（空部屋削除・ホスト交代）。プレイ中に切断した人は提出者数の
  分母から除外し、残りが全員提出済みなら結果発表に進む

## クライアント: `public/quiz.html`（新規1ファイル）

coin.html / order.html と同じデザイントークン（CSS変数・カード・ボタン）を踏襲。
スマホ前提のモバイルファースト。Lazywebで調査した参考例（Duolingo/Elevate/CapWords）の
パターンを反映する。

画面フロー（1ファイル内で画面切替）:

1. **参加/作成** — 名前＋ルームコード入力。`?mode=join` / `?mode=create` で初期表示切替
2. **ロビー** — 参加者一覧。ホストのみカテゴリ（Clacel/IELTS/TOEIC）→シリーズを選んで開始
3. **テスト画面** — 1画面1問。上部にプログレスバー（n/5）。日本語の意味＋頭文字ヒント＋
   穴埋め文＋大きな入力欄。「次へ」で進み、5問目は「提出する」。前の問題への「戻る」も可
   （提出までは answers をローカルに保持し、提出時にまとめて送る）
4. **待機画面** — 「提出済み n/総人数」を大きく表示
5. **ランキング画面** — 1位を大きくお祝い表示、以下順位表（名前・正解数・タイム）。
   自分の答案と正答の答え合わせ一覧も表示。「もう一度」でロビーに戻る（ホスト操作）

## トップページ変更: `public/index.html`

「単語テスト」カードの2ボタンを外部URLから `/quiz.html?mode=join` /
`/quiz.html?mode=create` に差し替え。説明文を「シリーズを選んで5問」の実態に合わせて更新。
Survive カードは外部リンクのまま触らない。

## エラー処理

- 存在しないルームコード / 開始済みルームへの参加 → 既存と同じエラーメッセージ
- 1人でも開始可能（練習用）。空の回答は不正解扱い
- サーバー再起動でルームは消える（既存2ゲームと同じ割り切り。永続化しない）

## テスト方針

- サーバーを起動し、ブラウザ2枚（ホスト＋参加者）で通しプレイ:
  作成→参加→シリーズ選択→開始→両者提出→ランキング表示まで確認
- 反証確認: 誤答・空欄提出で減点されること、存在しないルームコードでエラーが出ること、
  プレイ中の途中切断で残りメンバーだけで結果発表に進めること
- wordtests.js は `node -e` で読み込み、18セット×20問の件数と重複・空フィールドを機械チェック

## スコープ外（やらないこと）

- Survive の再実装
- 既存コインタワー/並べ替えバトルの変更
- Railwayへの再デプロイ設定（ローカル動作まで。デプロイは別途）
